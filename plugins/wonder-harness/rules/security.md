# 보안 규칙

## 시크릿

- 코드·템플릿·커밋에 시크릿(키·비밀번호·토큰)을 하드코딩하지 않는다.
- 환경변수 또는 시크릿 매니저를 사용한다.

## 입력 검증

- 모든 외부 입력(요청 파라미터, API 응답, 파일)은 처리 전 검증한다.
- 경계에서 실패하고 명확한 메시지를 남긴다 (fail-fast).

## 인젝션·XSS

- SQL 은 파라미터 바인딩만 사용한다 (문자열 연결 금지).
- 출력은 컨텍스트에 맞게 이스케이프한다 (HTML/속성/JS).

## 인증·인가

- 모든 보호 엔드포인트에 인가 검사를 둔다.
- 에러 메시지에 민감 정보를 노출하지 않는다.

## 검토 체크리스트 (review 모드)

- [ ] 하드코딩 시크릿 없음
- [ ] 입력 검증 존재
- [ ] 파라미터 바인딩 쿼리
- [ ] 인가 검사 존재
- [ ] 에러 메시지 정보 누출 없음
